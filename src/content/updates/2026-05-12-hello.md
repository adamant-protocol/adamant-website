Adamant is a Layer 1 blockchain protocol — privacy by default, sub-second finality, no foundation, no premine, no admin keys. The full pitch is on the [home page](/) and the plain-English version is in [About](/about).

This page exists for one reason: **it's better to have somewhere to point people at than not.**

## What's actually here

- A draft of the [whitepaper](/spec) — sections 0–2, 11, and the conclusion are inlined on the site; sections 3–10 are in the repo for public review.
- A full description of the [genesis mechanism](/genesis) — how the 100M ADM pool gets distributed, what doesn't happen.
- An [explainer for non-crypto people](/about) — no jargon, no acronyms.
- This [updates page](/updates) for everything as it lands.

## What's not here

There is no token. There is no test chain running. There is no allocation. There is no whitelist. There is no early access. There is no Discord with a "OG" role.

There will be a public testnet eventually. There will be independent audits before mainnet. Genesis happens when the work is ready — not when a marketing calendar says so.

## How to follow along

- The repo is at [github.com/adamant-protocol/adamant](https://github.com/adamant-protocol/adamant). All commits are public from day one.
- Open issues and discussions on GitHub if you've got something to say about the spec.

That's it for now. More soon.
